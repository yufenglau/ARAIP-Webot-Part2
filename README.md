# AI Integration in Robotics: Webots e-puck & CNN

## 📌 Project Overview
This repository contains the source code, machine learning models, and documentation for the ARAIP Coursework (Assignments 3, 4, & 5). The project demonstrates the integration of a custom Convolutional Neural Network (CNN) trained on the CIFAR-10 dataset with an autonomous `e-puck` robot in the Webots simulator. 

The core objective is to bridge the "reality gap" by allowing a resource-constrained robot to navigate, detect objects using an RGB pre-filter, and capture images for decoupled AI classification.

## 🚀 Key Features
* **Autonomous Navigation:** The e-puck utilizes 8 IR sensors for real-time obstacle avoidance.
* **RGB Pre-Filter Logic:** A lightweight mathematical filter (`DOMINANCE_MARGIN` and `DOG_TOLERANCE`) to detect targets without overloading the robot's CPU.
* **Finite State Machine:** Programmed in Python to handle searching, stopping, capturing `.png` payloads without motion blur, and evading.
* **Custom CNN Architecture:** A 4-layer Convolutional Neural Network built with TensorFlow/Keras, achieving 71% validation accuracy on CIFAR-10.
* **Edge Optimization Proposal:** A documented strategy for deploying the model using INT8 Post-Training Quantization and a decoupled Edge Server (Raspberry Pi).

## 📁 Repository Structure
* `/webots_controller` - Contains the Python control script (`capture_image.py`) for the e-puck robot.
* `/cnn_models` - Contains the Google Colab Jupyter Notebook (`.ipynb`) for training and testing the CNN.
* `/captured_data` - Contains the custom testing images and the outputs captured natively by the Webots camera.
* `/docs` - Contains project reports and presentation slides.

## 🛠️ Technologies Used
* **Simulation:** Webots (Cyberbotics)
* **Languages:** Python 3
* **Machine Learning:** TensorFlow, Keras, NumPy, Matplotlib
* **Environment:** Google Colab

## ⚙️ How to Run the Project
**1. Webots Simulation:**
* Open Webots and load the designated arena world file.
* Ensure the `e-puck` robot is assigned the Python controller located in the `/webots_controller` folder.
* Run the simulation. Captured images will be saved directly to the controller directory.

**2. CNN Inference:**
* Upload the notebook from `/cnn_models` to Google Colab.
* Run all cells to compile and train the model.
* Upload the images generated by Webots into the Colab environment to test real-world prediction accuracy.
